What is Vecode?

Vecode is an AI coding agent that runs entirely in your browser. You describe what you want to build in plain English, and the agent writes files, executes JavaScript in a sandbox, and shows you a live preview — all without installing anything.

It uses a ReAct (Reason + Act) loop: the AI thinks about your request, calls tools to write files and run code, observes the results, and iterates until the task is done.

30-second setup

  1. Click ⚙ Settings in the top bar.
  2. Click a provider chip (e.g. Groq (free) or OpenAI).
  3. Paste your API key. (No key? See the Free APIs tab.)
  4. Click Save Settings. The green dot in the logo means you're connected.
  5. Type a request like build a todo app with local storage and hit Enter.
No key yet? The Free APIs tab below lists providers that give you a key in under 60 seconds — Groq, HuggingFace, Cloudflare, and more — all free.

Bring Your Own Key (BYOK)

Vecode does not include a built-in API. You bring your own key from any OpenAI-compatible provider. This keeps you in full control of costs, model choice, and data privacy.

Where your key goes: It is stored only in your browser's localStorage. It is sent only to the endpoint URL you configure — never to any third-party server.

Common issues

  • 401 Unauthorized: key is wrong or has been rotated. Re-paste it.
  • 404 Not Found: the endpoint URL is wrong — it should end with /v1 (or /openai for Gemini).
  • Model not found: check the model ID matches exactly what the provider lists (e.g. llama-3.3-70b-versatile for Groq, not just llama3).
  • CORS error: some local providers (Ollama, LM Studio) need to be started with CORS enabled. For Ollama: OLLAMA_ORIGINS=* ollama serve.
  • Rate limit (429): free tiers are limited — wait a minute, or switch to a paid provider.
Tip: The show button next to the API key field lets you verify what you pasted. Keys are case-sensitive.

Picking a free provider

  • For speed: Groq. Llama 3.3 70B at 500+ tokens/sec.
  • For model variety: HuggingFace or OpenRouter (free models).
  • For privacy / offline: Ollama. Runs locally, no key, no internet.
  • For reliability: Cloudflare Workers AI — backed by Cloudflare's global network.
  • For multimodal (vision): Google Gemini — free tier includes image input.

The agent's tools

Vecode's AI uses these tools to actually do work — not just chat about it. You'll see them being called in real-time in the Console pane.

ToolWhat it does
write_fileCreates or overwrites a file in the virtual file system (VFS).
read_fileReads the content of a VFS file — used to verify its own work.
list_filesLists every file currently in the VFS.
execute_jsRuns JavaScript in a sandboxed Web Worker. Captures console.log, errors, and return values.
render_previewBundles HTML/CSS/JS into the live preview iframe on the right.
learn_skillPermanently saves a new skill to localStorage. Skill instructions are injected into the system prompt on every future turn. The agent gets better over time.
list_skillsLists all skills the agent has learned so far.
task_completeSignals that the task is done. The argument is the final summary you see.

The ReAct loop

  1. Reason — the AI plans what to do next.
  2. Act — it calls one or more tools.
  3. Observe — it reads the tool results from the console.
  4. Repeat — until it calls task_complete or returns a final text answer.

The loop is capped at 20 iterations to prevent runaway costs. If it hits the cap, you'll see a warning in chat — refine your prompt and try again.

Writing better prompts

  • Be specific: a pomodoro timer with a circular progress ring and sound alerts beats a timer.
  • State the audience: for a 5-year-old learning to count vs for an accountant produces very different UIs.
  • Iterate: after the first build, say now add a dark mode toggle or make the buttons bigger — the agent has the full VFS context.
  • Ask for explanations: explain how the auth flow works — the agent will read its own files and answer.

Choosing a model

  • Big models (gpt-4o, claude-3.5-sonnet, llama-3.1-405b): best for complex multi-file projects.
  • Mid models (gpt-4o-mini, gemini-1.5-flash, llama-3.3-70b): great default for most tasks — fast and cheap.
  • Small models (llama-3.1-8b, mistral-small, gemini-1.5-flash-8b): fine for simple components but may need more iterations.
  • Vecode's system prompt is tuned to make even small models reliable — it gives explicit step-by-step guidance and forces them to use tools.

Keyboard shortcuts

  • Enter — send message
  • Shift+Enter — newline in input
  • Esc — close any open modal

Export & import

  • Export → .zip: download all files as a zip archive (built-in ZIP encoder, no libraries).
  • Export → bundled HTML: one self-contained index.html with all CSS/JS inlined.
  • Export → JSON: save the whole VFS as a .json you can re-import later.
  • Import files from disk: pick one or more files (.html, .css, .js, .md, .py, anything text) and they're added to the VFS. Or just drag-and-drop files anywhere on the window.
  • Export / Import config code: a base64 string that captures your entire setup — theme, API settings, files, and skills. Great for backups or moving to another browser. Optionally include the API key (with a warning — anyone with the code can use it).

Auto-learned skills

The agent has a learn_skill tool. When it discovers a reusable technique, pattern, or design principle, it can save it as a permanent skill. Learned skills are injected into the system prompt on every future turn, so the agent gets better over time.

  • Automatic: ask the agent to "learn a skill for X" or just let it decide on its own.
  • Manual: add skills in Settings → Learned Skills. Give each one a name, short description, and full instructions.
  • Persistent: skills survive page reloads (stored in localStorage).
  • Portable: include them in a config code to share with another browser.

Themes

Vecode ships with 13 themes: Carbon, Slate, Solarized, Tokyo Night, Dracula, Gruvbox, Nord, Rosé Pine, One Dark, Catppuccin, GitHub Light, Synthwave, and Ayu Dark. Switch in Settings → your choice is saved.
